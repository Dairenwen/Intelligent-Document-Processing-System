
## 一、拉取（下载）仓库到本地

### 1. 找到仓库地址
- 打开队友发给你的GitHub仓库链接
- 点击绿色的 **"Code"** 按钮
- 复制HTTPS地址（例如：`https://github.com/队友名/仓库名.git`）

### 2. 克隆到本地
在命令行中执行：
```bash
# 进入你想存放项目的目录
cd D:\VueProj

# 克隆仓库
git clone https://github.com/队友名/仓库名.git
```
执行后会创建一个新文件夹，里面就是完整的项目文件。

---

## 二、修改代码并提交（标准流程）

### Step 1：创建新分支（重要！）
**永远不要直接在 `main` 分支上修改**，这是团队协作的基本原则。

```bash
# 进入项目目录
cd 仓库名

# 创建并切换到新分支（分支名建议包含你的名字缩写和功能，例如：zyh-fix-upload）
git checkout -b 你的分支名
```

### Step 2：进行修改
用VS Code打开项目，修改你需要改的文件。

### Step 3：查看修改状态
```bash
git status
```
会显示你修改了哪些文件。

### Step 4：添加修改到暂存区
```bash
# 添加所有修改的文件
git add .

# 或者只添加特定文件
git add 文件名
```

### Step 5：提交修改
```bash
git commit -m "清晰描述你的改动，例如：修复文件上传页的样式问题"
```
提交信息要写清楚做了什么，方便队友理解。

### Step 6：推送分支到GitHub
```bash
git push origin 你的分支名
```

---

## 三、创建Pull Request（PR）

推送成功后，按以下步骤请求合并你的代码：

1. 打开GitHub仓库页面，你会看到一个黄色的提示框，显示你的分支刚刚推送了修改，旁边有 **"Compare & pull request"** 按钮，点击它
2. 填写PR信息：
   - **标题**：简要说明改动
   - **描述**：详细说明改了哪里、为什么改
3. 点击 **"Create pull request"**

---

## 四、合并代码

PR创建后，**最好让队友来审查和合并**，不要自己合并自己的PR（除非紧急情况）。

- 队友可以查看你的代码，留下评论
- 如果需要修改，你在本地继续修改后，重复 `git add .` → `git commit` → `git push`，PR会自动更新
- 队友审核通过后，点击 **"Merge pull request"** 完成合并

---

## 五、保持本地代码最新

每次开始工作前，先拉取队友的最新修改：

```bash
# 切换到main分支
git checkout main

# 拉取最新代码
git pull origin main

# 切回你的工作分支（如果需要）
git checkout 你的分支名

# 将main的最新代码合并到你的分支
git merge main
```

## 六、常用命令速查表

| 操作 | 命令 |
|------|------|
| 克隆仓库 | `git clone 仓库地址` |
| 创建并切换分支 | `git checkout -b 分支名` |
| 查看修改状态 | `git status` |
| 添加所有修改 | `git add .` |
| 提交修改 | `git commit -m "说明"` |
| 推送分支 | `git push origin 分支名` |
| 拉取最新代码 | `git pull origin main` |
| 合并其他分支 | `git merge 分支名` |

---

## ⚠️ 注意事项

1. **永远不要在 `main` 分支上直接修改**，一定要新建分支
2. **提交前先拉取**：推送前最好先 `git pull origin main`，避免冲突
3. **提交信息要清晰**：让队友一眼看懂你改了啥
4. **如果遇到冲突**：Git会提示哪些文件冲突，需要手动打开文件解决（删除 `<<<<<<<`、`=======`、`>>>>>>>` 这些标记），然后重新提交
