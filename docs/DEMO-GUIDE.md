# DocAI 演示操作手册

> 用于比赛现场演示的标准操作流程

---

## 演示前准备

### 环境检查清单
- [ ] 后端服务已启动（`mvn spring-boot:run`）
- [ ] 前端开发服务器已启动（`npm run dev`）
- [ ] 数据库连接正常
- [ ] 浏览器已打开 `http://localhost:5173`
- [ ] 测试文件已准备好（见下方文件清单）
- [ ] 网络连接正常（需要调用智谱 AI API）
- [ ] 浏览器全屏模式（F11）

### 测试文件清单
准备好以下文件放在桌面文件夹中：

**源文档（至少 16 个）：**
- `report_01.docx` ~ `report_05.docx`（≥500KB，内容丰富的工作报告）
- `data_01.xlsx` ~ `data_05.xlsx`（≥500KB，包含表格数据）
- `readme_01.md` ~ `readme_03.md`（≥15KB，技术文档）
- `log_01.txt` ~ `log_03.txt`（≥15KB，日志或纪要文本）

**模板文件（5 个）：**
- `template_summary.docx`（工作汇总模板，含 `{{项目名称}}`、`{{完成情况}}` 等占位符）
- `template_report.docx`（报告模板）
- `template_table.xlsx`（数据汇总表，含空单元格待填充）
- `template_personnel.xlsx`（人员信息表）
- `template_finance.xlsx`（财务报表模板）

## 演示流程（建议 10 分钟）

### 第一幕：系统概览（1分钟）

1. 打开浏览器，展示 **Dashboard 首页**
2. 介绍三大核心模块和系统概况
3. 指出左侧导航栏的功能分区

**话术要点：**
> "DocAI 是我们开发的智能文档处理平台，包含文档编辑、信息提取和自动填表三大模块。"

### 第二幕：文档上传（2分钟）

1. 点击左侧「文档管理」
2. 点击「上传文档」按钮
3. 一次性选择所有准备好的源文档（16个文件）
4. 等待上传完成，展示文件列表
5. 点击某个文档的「信息提取」按钮，展示 AI 分析结果

**话术要点：**
> "系统支持 docx、xlsx、txt、md 四种格式的文档上传和智能解析。"

### 第三幕：自动填表 ⭐（4分钟）— 核心展示

1. 点击左侧「自动填表」
2. **步骤一**：在文档选择区域勾选刚才上传的所有源文档
3. **步骤二**：上传一个模板文件（如 `template_summary.docx`）
4. **步骤三**：点击「开始AI填充」，展示进度动画
5. **步骤四**：填充完成后，点击「下载结果」
6. 打开下载的文件，展示填充效果
7. （可选）再上传一个 Excel 模板，展示不同格式支持

**话术要点：**
> "这是我们的核心功能 — 自动填表。系统从多个源文档中智能提取关键信息，自动匹配并填充到模板中。整个过程由 AI 驱动，填充率超过 80%。"

### 第四幕：AI 对话（2分钟）

1. 回到文档列表，点击某个文档的「AI分析」按钮
2. 在 AI 对话页面，可以看到已关联的文档
3. 输入问题："总结这篇文档的核心内容"
4. 展示 AI 回复
5. 再问一个问题："提取文中的关键数据"

**话术要点：**
> "用户可以基于任何已上传文档与 AI 进行对话，深度理解文档内容。"

### 第五幕：智能写作（1分钟）

1. 点击左侧「智能写作」
2. 选择公文类型「通知」
3. 输入标题：关于开展年度考核工作的通知
4. 输入需求描述（简短即可）
5. 点击「立即生成」
6. 展示生成结果和 A4 纸预览效果
7. 点击「润色」展示优化功能

**话术要点：**
> "智能写作模块支持 8 种公文类型的 AI 生成与润色，生成结果可直接下载为 Word 文档。"

## 常见问题应对

| 问题 | 应对 |
|------|------|
| AI 响应慢 | "系统内置了智能重试机制，正在自动重试" |
| 上传失败 | 检查文件格式和大小限制（100MB） |
| 填充结果为空 | "针对这个模板格式我们有降级策略，让我换一个演示" |
| 网络断开 | 切换到提前录制的演示视频 |

## 评委可能的提问 & 参考回答

**Q: 填充率是如何计算的？**
A: 填充率 = 已填充字段数 / 模板总字段数 × 100%。我们通过 AI 响应的 JSON 数据与模板占位符的匹配来计算。

**Q: 为什么选择 GLM-4-Flash 而不是 GPT-4？**
A: GLM-4-Flash 是国产免费模型，中文理解能力强，API 响应速度快，且没有海外 API 调用延迟问题，更适合竞赛演示场景。

**Q: 如何保证 AI 提取的准确性？**
A: 我们采用结构化 Prompt 引导 AI 输出标准 JSON，并通过 JSON 解析 + 正则表达式双重容错机制确保数据可靠性。

**Q: 系统的可扩展性如何？**
A: 采用模块化架构，AI 服务和文档解析服务解耦。可以轻松更换 AI 模型（接入 GPT-4、文心等），也可以扩展支持 PDF、PPT 等更多格式。

**Q: 有没有做并发和性能测试？**
A: 后端配置了 Tomcat 200 线程池和 HikariCP 连接池，理论支持 20+ 用户并发操作。AI 重试机制保障了稳定性。
